# Portfolio Summary

## Project

Yandex Cloud Security Remediation Program

## Short description

Built a production-like cloud security remediation program on real managed cloud infrastructure using Yandex Cloud, Terraform, IAM/OIDC federation, SAML/SSO federation patterns, Managed Kubernetes hardening, policy-as-code, CI/CD security gates, audit logging and evidence-driven before/after remediation metrics.

## Russian resume version

Реализовал production-like cloud security remediation program на реальной managed cloud infrastructure Yandex Cloud: Terraform IaC, IAM/OIDC federation для GitHub Actions без долгоживущих ключей, SAML/SSO federation pattern, Managed Kubernetes hardening, policy-as-code, CI/CD security gates, audit logging и evidence-based remediation. Проект опубликован как воспроизводимый GitHub-кейс с sanitized evidence package, before/after metrics, risk register и cloud-control mapping на AWS/GCP/Azure.

## What makes this project strong

This project is not only a Kubernetes lab. It connects architecture, IaC, identity, CI/CD, Kubernetes hardening, policy-as-code, audit evidence, measurable remediation and safe GitHub publication into one coherent security engineering case.

## Boundaries

The project is implemented on Yandex Cloud due to payment and access constraints.

AWS, GCP and Azure are used only for provider-agnostic control mapping. This project must not be described as production experience with AWS, GCP or Azure.
