# Yandex Cloud readiness gate

## Назначение

Эта фаза проверяет, что локальная рабочая среда готова к будущему Yandex Cloud run.

Проверяются только read-only условия:

- доступность yc CLI;
- наличие cloud-id;
- наличие folder-id;
- возможность прочитать cloud;
- возможность прочитать folder;
- наличие Terraform example variables;
- отсутствие state, kubeconfig и ключевых файлов в репозитории.

## Граница фазы

В этой фазе не выполняются операции создания, изменения или удаления ресурсов.

Cloud run разрешается только в отдельной последующей фазе после явного подтверждения.
